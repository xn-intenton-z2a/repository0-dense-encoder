Purpose
- RFC 1924 proposes a compact single-token base85 representation for 128-bit IPv6 addresses, mapping the full 128-bit value into 20 printable characters (20*log2(85) ≈ 128).

Encoding base and bit density
- Base: 85
- Bits per character: log2(85) ≈ 6.409391 bits/char
- For 128-bit values: 128 / 6.409391 ≈ 19.976 -> 20 characters required.

Canonical alphabet (RFC-specified ordering)
- RFC 1924 defines an ordered 85-character alphabet to be used when encoding 128-bit IPv6 values. Implementations must use the exact ordering from the RFC to interoperate.
- For implementation, read the RFC section that enumerates the 85-character table and apply it as a mapping index -> character when converting base-85 digits to output characters.

Encoding algorithm (implementation-ready)
- Interpret the IPv6 128-bit address as an unsigned integer N (most-significant octet first).
- Repeatedly divide N by 85 to produce 20 base-85 digits d0..d19 where d0 is the most-significant digit:
  - For i = 19 down to 0: digits[i] = N % 85; N = floor(N / 85)
  - After loop digits[0..19] represent the value in base-85 least-significant digit at digits[19].
- Map each digit to the RFC's alphabet and emit characters d0..d19 in order.
- Decoding reverses: map characters to digits, compute N = sum(d[i] * 85^(19-i)), then split into 16 bytes big-endian.

UUID length and comparison note
- For arbitrary 128-bit binary values such as a UUID, the RFC1924 base85 mapping yields 20 characters (same as Ascii85 for 16 bytes) and therefore is denser than base64 without padding (22 chars).

REFERENCE DETAILS
- API signatures (exact):
  encode128(input: Uint8Array[16]): string    # returns 20-char base-85 string using RFC alphabet
  decode128(input: string): Uint8Array[16]    # accepts exactly 20 characters from the RFC alphabet
- Validation rules:
  - encode128 requires exactly 16 bytes input.
  - decode128 must accept exactly 20 characters and must reject any character not in the RFC alphabet.

SUPPLEMENTARY DETAILS
- Implementation: use arbitrary-precision integer arithmetic (128-bit) or operate on 16-byte arrays and perform repeated division by 85 with long-division to avoid big-int dependencies.
- Interoperability: must use the RFC's exact alphabet ordering; different orderings produce different encoded strings.
