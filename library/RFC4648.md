Normalised extract (direct technical details)
Core encoding model
- Encodings operate by grouping input octets into groups of 3 (24 bits) and dividing those 24 bits into four 6-bit values.
- Each 6-bit value indexes into a 64-character alphabet (values 0..63).
- Standard Base64 alphabet (RFC 4648 Base 64): A-Z (0..25), a-z (26..51), 0-9 (52..61), + (62), / (63).

Alphabet and variants
- Standard Base64 alphabet:ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
- URL-safe Base64 (base64url) replaces + with - and / with _ and may omit padding.
- Padding: standard Base64 uses '=' characters to pad output to a multiple of 4 characters; padded output length = 4 * ceil(n/3).

Encoding and decoding rules
- For each 3 input bytes (b0,b1,b2): produce four output chars whose indices are:
  - c0 = (b0 >> 2) & 0x3F
  - c1 = ((b0 << 4) | (b1 >> 4)) & 0x3F
  - c2 = ((b1 << 2) | (b2 >> 6)) & 0x3F
  - c3 = b2 & 0x3F
- If input length modulo 3 != 0, produce appropriate c0..c2 and pad with '=' characters when using padded mode.
- Decoding reverses these operations; decoder must handle optional padding and reject invalid characters.

Length formulas
- Padded output length = 4 * ceil(input_length / 3).
- Padding count = (3 - (input_length % 3)) % 3; padded characters are '=' repeated padding count times when using padded mode.
- Unpadded length = padded length - padding_count (if '=' removed).
- Example: For a UUID (16 bytes): padded length = 4 * ceil(16/3) = 4 * 6 = 24; input_length % 3 = 1, padding count = 2; unpadded length = 24 - 2 = 22.

No-padding variant and UUID baseline
- For compact representation omit padding '=' characters (commonly used for URL-safe or compact encodings). When omitting padding the exact output length is determined by input length and algorithm above.
- For 16 bytes (UUID) the no-padding base64 length is 22 characters; this is the baseline the mission references.

Reference API signatures
- encodeBase64(input: Uint8Array, options?: {urlSafe?: boolean, padding?: boolean}) -> string
  - urlSafe: boolean, when true use '-' and '_' instead of '+' and '/'
  - padding: boolean, when false do not append '=' padding
- decodeBase64(input: string, options?: {urlSafe?: boolean}) -> Uint8Array
  - Accepts padded or unpadded input; when unpadded, decoder should compute required padding for processing.

Supplementary implementation details
- For high-performance, operate on 4-character output chunks and use table lookup for 6-bit -> char mapping.
- Avoid string concatenation in hot loops; collect into array of bytes/indices and map to characters at the end.
- Be careful with signed byte arithmetic in languages with signed bytes; use unsigned masking (0xFF) when shifting.
