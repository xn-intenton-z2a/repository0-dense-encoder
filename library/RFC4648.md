NORMALISED EXTRACT

Base encodings defined: Base16 (hex), Base32, Base64 and the base64url variant. Core encoding rule (Base64):
- Take input bytes as a sequence of bits in big-endian order.
- Group bits into 6-bit units from most-significant to least-significant within the input stream.
- Map each 6-bit value (0..63) to the Base64 alphabet: "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/".
- Output is the concatenation of mapped characters; if the final quantum of input bits is not a multiple of 24 bits, pad the input with zero bits and use '=' characters in the output so the output length is a multiple of 4. The no-padding variant omits '=' and requires decoder to infer missing bits.

Base64url variant:
- Alphabet: same ordering except characters 62 and 63 are '-' and '_' respectively instead of '+' and '/'.
- Padding: optional. Many usages omit '=' padding for compactness; decoder must handle missing padding.

Quantitative properties:
- Bits per character: Base64 = 6.0 bits/char
- For 16-byte input (UUID, 128 bits): ceil(128 / 6) = 22 characters (no padding).

DETAILED IMPLEMENTATION (encoder/decoder rules)
- Encoding algorithm (deterministic steps): read input bytes into a 24-bit buffer (3 bytes), emit four 6-bit indices: index0 = (b0 >> 2), index1 = ((b0 & 0x03) << 4) | (b1 >> 4), index2 = ((b1 & 0x0F) << 2) | (b2 >> 6), index3 = b2 & 0x3F; map indices to alphabet. For final partial block of 1 or 2 bytes, pad with zero bits and emit 2 or 3 characters plus '=' padding when padding enabled.

- Decoding algorithm: accept characters in the alphabet and optionally handle '-'/'_' mapping for base64url; replace missing padding by appending '=' or reconstruct bits by bit-accumulator technique; validate that output byte length matches expected length.

REFERENCE DETAILS (API signatures)
- base64Encode(bytes: Uint8Array, urlSafe: boolean = false, padding: boolean = true): string
  - urlSafe true => use '-' and '_'
  - padding false => omit '=' from output

- base64Decode(str: string, urlSafe: boolean = false): Uint8Array
  - Accepts optional omitted padding; rejects characters outside the alphabet.

BEST PRACTICES
- For strict round-trip, use standard alphabet with padding. For compact representations (UUIDs), use urlSafe=true and padding=false to produce 22-character output for 16 bytes.
- Avoid using platform btoa/atob for arbitrary bytes; convert bytes to binary string correctly or use native Buffer.from(...).toString('base64') in Node.
