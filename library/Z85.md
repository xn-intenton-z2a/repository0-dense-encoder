NORMALISED EXTRACT:
Alphabet (Z85 exact order): 0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ.-:+=^!/*?&<>()[]{}@%$#
Charset size: 85 (bits per char = log2(85) ≈ 6.409 bits/char)
Block geometry: Z85 encodes 4 bytes (32 bits) into 5 characters and decodes groups of 5 characters into 4 bytes. Implementations MUST operate on frames whose binary length is divisible by 4 and whose encoded string length is divisible by 5. Padding is an application-level responsibility.
Encoding algorithm (formal):
- Treat each 4-octet block as an unsigned 32-bit integer in network byte order (big-endian).
- Produce five base-85 digits by repeatedly dividing by 85, output characters from most significant to least significant using the alphabet index mapping shown above.
Decoding algorithm (formal):
- For each 5-character group, convert characters to their 0..84 index values, accumulate value = (((((v0)*85 + v1)*85 + v2)*85 + v3)*85 + v4).
- Extract four bytes from the 32-bit big-endian integer and append to output.
Interoperability notes:
- Z85 explicitly avoids special-case contractions (no 'z' shorthand) and selects characters safe for embedding in source code and shell quoting.
- Implementations must validate string length mod 5 == 0 and reject invalid characters.

REFERENCE DETAILS (implementable signatures):
- encode(bytes: Uint8Array): string
  - Requirement: input length % 4 == 0 or application pads the input before calling encode.
- decode(str: string): Uint8Array
  - Requirement: str length % 5 == 0.
