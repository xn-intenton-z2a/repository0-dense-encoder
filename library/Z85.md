DETAILED INFORMATION

Exact alphabet (index -> character, ordered 0..84):
0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ.-:+=^!/*?&<>()[]{}@%$#

Bits per character: log2(85) = 6.409394 bits/char.

Frame rules (mandatory for interoperability):
- Binary frames MUST have length divisible by 4 (no remainder).
- String frames MUST have length divisible by 5.
- Applications must pad binary data to 4-byte multiple or use a framing protocol that carries length.

Encoding algorithm (exact steps):
- For each 4-byte big-endian block produce unsigned 32-bit V.
- Compute five digits d0..d4 where V = (((d0 * 85) + d1) * 85 + d2) * 85 + d3) * 85 + d4.
- Output characters alphabet[d0]..alphabet[d4].

Decoding algorithm:
- For each 5-character group map chars to values 0..84 and compute V by repeated multiply-add by 85; split V into 4 bytes big-endian.
- Strictly enforce frame length divisibility and reject invalid characters.

Interoperability notes:
- Z85 removes Ascii85 ambiguities (no 'z' shorthand) and selects an alphabet safe for inclusion in source code and URLs where possible.

REFERENCE DETAILS (API signatures)

encodeZ85(bytes: Uint8Array): string
 - Requires bytes.length % 4 === 0 (or application pads prior to call)
decodeZ85(str: string): Uint8Array
 - Requires str.length % 5 === 0; returns decoded bytes

Implementation notes:
- Use 32-bit unsigned arithmetic to construct V to avoid floating-point issues.
- For partial use-cases pad to multiple of 4 and transmit original length via separate channel or prefix.
