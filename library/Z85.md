Z85

NORMALISED EXTRACT

Z85 is a base-85 encoding variant designed for use in source code and other textual contexts; it encodes every 4 bytes into 5 printable characters. Key deterministic rules:
- Input byte-frame length MUST be divisible by 4. String output length is therefore divisible by 5.
- Each 4-octet big-endian unsigned integer is converted to a 5-digit base-85 representation and mapped to a printable ASCII character set.

Exact Z85 alphabet (index 0 .. 84, in order):
0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ.-:+=^!/*?&<>()[]{}@%$#

Algorithmic details (precise):
- Encode: for every 4 bytes, form a 32-bit unsigned integer N (network order). Compute values v4..v0 such that N = v0*85^4 + v1*85^3 + v2*85^2 + v3*85 + v4, then output characters alphabet[v0]..alphabet[v4].
- Decode: take groups of 5 characters, map each char to index 0..84, accumulate N = ((((i0*85)+i1)*85+i2)*85+i3)*85+i4, then split N into 4 output bytes in big-endian order.

NUMERICAL PROPERTIES
- Bits per character: log2(85) ≈ 6.409 bits/char
- Charset size: 85
- For 16 bytes: exact encoded length = (16 / 4) * 5 = 20 characters.

TOC

SUPPLEMENTARY DETAILS
- Z85 intentionally selects characters that are safe to embed in source code and configuration formats, and avoids whitespace and ambiguous characters.
- Implementation must enforce that strings contain only the 85-character alphabet; reject control characters and unexpected punctuation.

REFERENCE DETAILS (API signatures)
- z85Encode(bytes: Uint8Array): string
  - Input length must be a multiple of 4; callers should pad prior to encoding or use a container format.

- z85Decode(str: string): Uint8Array
  - Input length must be a multiple of 5; decode into bytes and return Uint8Array.

BEST PRACTICES
- For UUID (16 bytes) produce 20 characters exactly (no padding required) by encoding the 16 bytes directly as four 4-byte frames.
- Use strict validation to ensure round-trip property: z85Decode(z85Encode(bytes)) === bytes.
